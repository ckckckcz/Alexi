Apa Itu Framework Bootstrap ? Pengertian, Sejarah, Kelebihan, Kekurangan, dan Bagaimanan Cara Penggunaanya

Bootstrap membuat front-end developer dapat membuat website dengan cepat, fokus pada responsive mobile, dan membuat website menjadi lebih interaktif tanpa membuat banyak CSS dan JavaScript dari nol.

* Pengertian Framework Bootstrap

Bootstrap adalah framework CSS, yang berfungsi untuk mendesain website responsive dengan cepat dan mudah.

Bootstrap, awalnya bernama Twitter Blueprint, dikembangkan oleh Mark Otto dan Jacob Thornton di Twitter. Sebelum Bootstrap, berbagai pustaka sudah banyak digunakan untuk pengembangan antarmuka, yang menyebabkan ketidak-konsistenan dan beban pemeliharaan yang berat.

* Sejarah Framework Bootstrap

Bootstrap 1


Menurut Mark Otto ( pengembang Twitter ), Setelah beberapa bulan pengembangan oleh sebuah kelompok kecil, banyak pengembang di Twitter mulai memberikan kontribusi untuk proyek ini sebagai bagian dari Hack Week, sebuah pekan semacam HackAthon untuk tim pengembangan di Twitter. Kemudian nama yang awalnya Twitter Blueprint diganti menjadi Bootstrap, dan dirilis sebagai proyek open source pada 19 agustus 2011.


Bootstrap 2 dan 3


Pada tanggal 31 Januari 2012, Bootstrap 2 dirilis yang dimana bootstrap menambahkan sistem layout dua belas kolom, grid yang Responsive, dukungan built-in untuk Glyphicons, beberapa komponen baru, serta perubahan dari beberapa komponen yang sudah ada.

Pada 19 Agustus 2013, Bootstrap 3 dirilis, yang didesain ulang komponennya untuk menggunakan desain flat, dan pendekatan pada mobile-first.


Bootstrap 4


Versi alpha pertama dari Bootstrap 4 dirilis pada 19 Agustus 2015. Kemudian pada tanggal 6 September 2016, Mark menghentikan pengerjaan Bootstrap 3 dalam rangka untuk meluangkan waktu untuk pengerjaan Bootstrap 4. Sejauh ini, sudah lebih dari 17.500 kommit yang telah dibuat untuk basis kode Bootstrap 4. Versi beta pertama dirilis pada 10 Agustus 2017.

Versi stabil dari Bootstrap 4 dirilis pada 18 Januari 2018 , mayoritas merupakan tulisan ulang dari versi sebelumnya yaitu Bootstrap 3. Perubahan yang signifikan di antaranya sebagai berikut :

- Peralihan dari Less menuju Sass.

- Peluncuran but ulang untuk normalisasi css.

- Menghapus dukungan peramban web IE8, IE9, dan IOS 6.

- Penambahan dukungan flexbox dan menghapus dukungan komponen non-flexbox.

- Penambahan kustomisasi pilihan navigasi.

- Penambahan utilitas untuk pemberian ruangan dan ukuran yang responsive.

- Beralih dari piksel (px) ke em.

- Meningkatkan ukuran font global dari 14px menjadi 16px.

- Menghapus panel, keluku, dan juga komponen-komponen yang mendukung element cards.

- Menghapus font ikon Glyphicons.

- Menghapus komponen pembuat halaman (pager).

- Menambahkan sejumlah besar utilitas class="".

- Memperbaiki perancangan formulir.

- Memperbaiki tombol dan dropdown.

- Memperbaiki objek media dan class untuk gambar.

- Penulisan ulang hampir smeua komponen, plugin jQuery, dan dokumentasi.


Bootstrap 5


Bootstrap 5 adalah versi baru Bootstrap yang akan datang. Perubahan besar termasuk :

- Menghapus jQuery dari komponen dan menggunakan Vanilla Javascript.

- Menulis ulang komponen grid untuk mendukung fitur kolom yang ditempatkan di luar baris dan gutter (ruang antar kolom) yang responsive.

- Migrasi perangkat lunak dokumentasi dari Jekyll ke Hugo.

- Menghapus dukungan peramban web IE10.

- Memindahkan infrastruktur pengujian dari QUnit ke Jasmine.

*Kelebihan Bootstrap


1. Mudah Dipakai Bagi Pemula

Meskipun baru memahami dasar HTML, CSS, dan JavaScript, kamu sudah bisa menggunakan Bootstrap untuk membuat website sederhana. Dengan alasan, ada banyak element dan class yang tersedia dan bisa langsung kamu pakai. Inilah mengapa Bootstrap sangat mudah dipakai untuk pemula.


2. Memiliki Grid Sistem yang Sederhana

Dengan adanya grid sistem ini untuk membuat website yang responsive sekarang menjadi hal yang mudah. Semua element yang didalam website secara otomatis akan menyesuaikan dengan device user gunakan. Dan juga, progammer juga akan dimudahkan karena sudah tidak perlu lagi membuat website versi mobile secara terpisah.


3. Bersifat Open-Source

Framework satu ini ( Bootstrap ) merupakan salah satu framework open-source yang memiliki kebebasan untuk dikembangkan oleh user lain. Hal ini juga membuat Bootstrap lebih berkualitas berkat berbagai konstribusi dari pengembang di seluruh dunia.


4. Kebebasan Kostumisasi

Versi default Bootstrap menyediakan semua yang kamu butuhkan untuk membuat website. Namun, bisa juga kamu mengubahnya dengan sesuai kebutuhan.


5. Rutin Update

Yang perlu kamu ketahui menggunakan framework yang selelu update itu penting. Dan salah satunya Bootstrap yang termasuk framework dengan update / pembaruan rutin.


6. Dokumentasi Yang Lengkap

Bagi developer dokumentasi itu sangat lah penting dan salah satunya Bootstrap yang memiliki dokumentasi yang sangat lengkap sehingga dapat memudahkan user yang ingin mempelajari / menggunakan framework tersebut dengan sangat mudah


7. Memiliki Komunitas Yang Besar dan Aktif

Jika kamu mengalami kesulitan saat belajar / menggunakan framework Bootstrap dan memiliki kendalan / error yang tidak bisa kamu pecahkan bahkan didalam dokumentasi tersebut tidak ada solusi ?

Tenang mas bro, komunitas bootstrap di seluruh dunia bisa membantu kamu. Para komunitas bootstrap sering kali membantu menjawab bermacam pertanyaan saat ada error dari berbagai developer pemula diseluruh dunia. Kamu bisa menanyakan melalui Github atau Stack Overflow.

*Kekurangan Bootstrap


1. Elementnya Banyak Digunakan Oleh Developer

Bootstrap merupakan framework yang menyediakan element cukup banyak. Namun, ada kalanya beberapa user menggunakan element dari bootstrap yang sama untuk kebutuhan website mereka. Dan akhirnya, tampilan website mereka mirip dengan beberapa website lain CSS


2. Bisa Memperlambat Website

Ketika kamu pertama kali di download, ukuran file Bootstrap memang tidak terlalu besar. File ini berisi CSS, JavaScript, hingga jQuery yang akan digunakan untuk membuat sebuah website mobile secara terpisah.

Seiring berjalannya waktu dan bertambahnya file di website tersebut, pada akhirnya membuat website menjadi berat. Solusinya, usahakan kamu hanya menggunakan class Bootstrap yang memang kamu perlukan saja.


3. Memperlambat Proses Belajar Coding

Jika kamu sedang memulai belajar coding, mengandalkan Bootstrap saja bukannlah pilihan yang bijak. Lah kenapa begitu ?,

Begini dengan kemudahan yang disediakan oleh Bootstrap, kamu bisa aja membuat website tanpa harus melakukan coding yang rumit, tidak bisa menghadapi error / bug, dan kamu tidak akan bisa mengalami gabisa tidur karena ada error pada code :").

Bahkan dengan code yang sudah disediakan pada halaman dokumentasi bootstrap, bisa saja kamu membuat sebuah website tanpa harus mempelajari konsep dasar


* Cara Penggunaan Bootstrap


Kali ini kita akan menggunakan Bootstrap dari CDN JSDeliver. Dengan demikian, kamu bisa langsung menggunakan class dari Bootstrap seperti yang dicontohkan pada tag <head>
